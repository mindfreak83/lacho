Entendiendo HTML, XHTML y HTML5

HTML es un acrónimo para "Hypertext Marku0 Language", el cuál es el lenguaje que usamos para crear páginas web desde que las primeras páginas llegaron a la web. Las raíces de HTML vienen de un antiguo lenguaje de modelado que se usaba para publicar documentos llamado SGML (Standard Generalizated Markup Language). Al igual que SGML, HTML usa etiquetas para dar formato tal como "<b>esto está en negritas</b>" el cual causará que el texto dentro de la etiqueta "b" se traduce a texto en negrita. Notece la diferencia entre la etiqueta de apertua y la etiqueta de cierre, la segunda etiqueta tiene una barra inclinada (/) para indicar que es una etiqueta de cierre. Muchos pero no todas las etiquetas HTML deben llevar la etiqueta de cierre. Etiquetas de HTML tales como <br> y <img> no tiene etiquetas de cierre, porque la etiquet <br> solo interpreta un salto de línea y la etiqueta <img> solo muestra una imagen.

Un interesante aspecto de HTML y su relación con los navegadores es que estos fueron diseñados para ser compatibles con versiones anteriores y con versiones recientes. Creando un navegador que es compatible con versiones anteriores es fácil, por que conoces el problema. Pero, cómo es la compatibilidad con versiones recientes? Los navegadores fueron creados para ignorar las etiquetas que no reconocen. Por ejemplo, si un navegador se encuentra con con una etiqueta <xyz> que no reconoce, debería de ignorar la etiqueta como si no existiera.

Aunque HTML sirve a su propósito por cierto tiempo, las personas buscan más. Otra tecnología envolvente llamada XML (eXtensible Markup Language), se volvió popular. XML es parecido a HTML por que ambos lenguajes usan etiquetas. Sin embargo, hay varias diferencias. Primero, XML siempre requiere que se cierren cada una de las etiquetas, aunque puedes usar una notación corta para denotar el inicio y el cierre de la etiqueta. Otra diferencia es que HTML tiene una colección definida de etiquetas y cada una de estas etiquetas denota una característica de formato que es aplicada a la página web que se presenta. XML no tiene una colección definidas de etiquetas, así que puedes crear las tuyas propias y las etiquetas pueden representar lo que sea. Las etiquetas XML son típicamente "metadata tags": etiquetas que que describen los datos que contienen. Aunque hay muchas otras diferencias, otra gran diferencia es que XML usa la tecnología de XML Schema Definition (XSD), la cual valida el formato de un documento XMLpara asegurarse que todos los aspectos de un documento son válidosantes de procesarel documento XML. La falte de rigidez en el archivo HTML ha prevenido la creación de una tecnología como XSD que pudiera validar documentos HTML.

El World Wide Web Consortium también conocido como W3C, es responsable de desarrollar estandares amplios para la web. El W3C ha introducido XHTML para solventar los problemas en HTML, el cual estaba en su versión 4. El XHTML es un XML-basado en la especificación de HTML para hace rque el HTML adicionara las reglas de XML que describe un bien estructurado documento, el cual debe tener una etiqueta de cierre por cad etiqueta de apertura. Esto significa que los documentos XHTML pueden ser validados por usar archivos XSD y pueden ser editados por usar herramientas XML.

Aunque el XHTML resuelve algunos problemas, otros problemas aún necesitan solución. Hubo necesidad por un incremento en la cantidad de multimedia en la web. Las compañías buscan los website mas llamativos. Los Cascading Style Sheet (CSS) proveen soporte para agregar estilos tales como colores y fuentes consistentes a través del website, pero las empresas buscan más. Ellos buscan que sus páginas web sea altamente interactivas, con video y animaciones. Los navegadores agregaron soporte programable para proporcionar Javascript, pero primeras versiones de Javascript eran lentas y difíciles de programar. Los navegadores se volvieron extendidas para proveer una interfaz de programación de aplicaciones (API) que permite a terceros crear plug-ins que puedan ejecutarse en el entorno del navegador. Un ejemplo de plug-in es Flash, "which has a very large installed base". Flash proporciona un entorno de desarrollo que puede usarse para crear una experiencia rica para el usuario. Aunque plug-ins de terceros resuelven el problema de la necesidad de tecnología para crear sitios web llamativos, existe la necesidad estricta de integrar multimedia con el navegador, especialmente con dispositivos pequeños.

HTML5 no se originó de XHTML, HTML5 se originó de HTML4.01. Como regla, sin embargo, aplicar reglas XHTML a HTML5 hará tu página web mas complaciente con una gran variedad de navegadores y lectores de páginas web, generadores y validadores. Este libro intenta ser lo mas complaciente con la mayoría de las tecnologías.

HTML5 representa una reinvención de HTML, CSS y Javascript de una forma que soluciona la necesidad de sitios web interactivos ricos que pueden reproducir audio y video y soportar animaciones desde el navegador sin la necesidad de plug-ins. HTML5 contiene la mayoría de las etiquetas de HTML4.01 pero la mayoría de estas etiquetas han sido redefinidas para convertirse en etiquetas semnánticas.

Introducción al modelado semántico

HTML5 enfatiza en separar la estructura, presentación y el comportamiento, una buena práctica para entender semántica es definida como el estudio del siginificado de las expresiones lingüísticas. En el contexto de HTML, significa que las etiquetas proveen significado al contenido en el documento HTML. Las etiquetas no proporcionan presentación, ellas proporcionan significado.

Las etiquetas HTML proporcionan estructura significativa, pero no así presentación. Recuerda que la separación es complemento para proporcionar estructura en el documento HTML5, mantener la presentación en la hoja de estilo CSS3 y mantener el comportamiento en el archivo Javascript.
